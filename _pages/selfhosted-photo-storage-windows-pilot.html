01 — Architecture

How It Works on Windows

On Linux, Docker runs natively. On Windows, Docker Desktop uses WSL2 — a real Linux kernel built into Windows — to run Linux containers transparently. From Immich's perspective, it's running on Linux. You never need to touch WSL2 directly.

Your Windows Laptop
🪟 Windows 11 (Host OS)
↕ built-in virtualisation
🐧 WSL2 (Linux kernel)
managed by Docker Desktop
↕
🐳 Docker Desktop
↕ containers
📸 Immich + PostgreSQL + Redis
↕ bind mount
💾 Windows folder (e.g. D:\Photos)
ℹ
You do not need to install a separate Linux VM. WSL2 is already built into Windows 10 (v2004+) and Windows 11. Docker Desktop sets it up automatically.
02 — Before You Start

Prerequisites & Honest Caveats

System requirements

RequirementMinimumRecommended
Windows version Windows 10 v2004 (build 19041) Windows 11
RAM 8 GB total (4 GB free for Docker) 16 GB
CPU Any modern dual-core Quad-core+, for face detection
Storage for photos Any folder / external drive Dedicated external HDD/SSD
Virtualisation Must be enabled in BIOS (usually is by default)

Pilot-mode trade-offs to be aware of

⚠️ Laptop-specific limitations
Immich is only reachable when the laptop is on and awake. Sleep = server offline.
Performance is shared with your normal laptop workload.
Docker Desktop on Windows uses slightly more RAM than a native Linux host (~200–400 MB overhead for WSL2).
If the laptop moves around, its local IP may change — use Tailscale to avoid this.
Everything you set up here migrates cleanly to dedicated hardware later.
No extra cost, no extra device.
03 — WSL2

Enable WSL2

Docker Desktop installs WSL2 automatically, but it's smoother to enable it first yourself. Open PowerShell as Administrator and run:

// PowerShell — Run as Administrator
wsl --install

This installs WSL2 and Ubuntu in one command. Restart your computer when prompted.

ℹ
If you see "WSL is already installed", just make sure you're on WSL2 by running wsl --set-default-version 2. You don't need to use the Ubuntu terminal at all — Docker Desktop takes it from here.

Verify virtualisation is active (it almost certainly is):

// PowerShell
systeminfo | findstr "Hyper-V Requirements"
# Should say: "A hypervisor has been detected"

If virtualisation is not detected, you need to enable it in your BIOS/UEFI settings. Search your laptop model + "enable virtualisation BIOS" for specific steps.

04 — Docker Desktop

Install Docker Desktop

01

Download Docker Desktop for Windows

Go to docker.com/products/docker-desktop and download the Windows installer. Run it and accept defaults. It will automatically configure WSL2 as its backend.

02

Configure Docker Desktop settings

After installation, open Docker Desktop → ⚙️ Settings and apply these:

  • General → Start Docker Desktop when you log in ✓
  • Resources → Memory: set to at least 4096 MB (4 GB)
  • Resources → CPUs: set to at least 2
  • WSL Integration → Enable for your Ubuntu distro ✓
03

Verify Docker is working

Open PowerShell (regular, not admin) and run:

docker --version
docker compose version

Both should print version numbers. If they do, you're ready.

05 — Immich

Deploy Immich

01

Create a folder for Immich config

Open PowerShell and create a working directory:

mkdir C:\immich
cd C:\immich
02

Download the official Immich files

Run these commands in PowerShell to download the Docker Compose file and environment config from Immich's official release:

# Download docker-compose.yml
Invoke-WebRequest -Uri "https://github.com/immich-app/immich/releases/latest/download/docker-compose.yml" -OutFile "docker-compose.yml"

# Download the .env template
Invoke-WebRequest -Uri "https://github.com/immich-app/immich/releases/latest/download/example.env" -OutFile ".env"
03

Edit the .env file

Open C:\immich\.env in Notepad (or VS Code). Find and update these two lines:

# Where your photos will be stored — point this to your drive
UPLOAD_LOCATION=C:\Photos\immich-library

# Where the database will be stored
DB_DATA_LOCATION=C:\Photos\immich-db
⚠
If you're using an external drive (e.g. D:\Photos), that drive must be plugged in every time Immich starts. If it's missing, Docker will throw an error. Consider using a folder on your main drive for the pilot.
04

Start Immich

cd C:\immich
docker compose up -d

First launch downloads all the container images (~2–3 GB). This takes a few minutes depending on your internet speed. Subsequent starts are instant.

05

Open Immich in your browser

http://localhost:2283

Create your admin account on first launch. Use a strong, unique password. You can also access it from other devices on your home network using your laptop's local IP:

# Find your local IP in PowerShell:
ipconfig
# Look for "IPv4 Address" under your active adapter
# e.g. 192.168.1.105

# From another device on your home WiFi:
http://192.168.1.105:2283
06

Enable 2FA in Immich

After logging in, go to Account Settings → Security → Enable two-factor authentication. Use an authenticator app (Google Authenticator, Authy, or Bitwarden). This is critical since you'll be exposing this to the internet.

Managing Immich (start / stop / update)

// PowerShell — run from C:\immich
# Stop Immich
docker compose down

# Start Immich
docker compose up -d

# Update to latest version
docker compose pull
docker compose up -d

# Check if containers are running
docker compose ps
✓
Docker Desktop starts automatically with Windows (if you enabled that setting), and it will restart the Immich containers automatically on boot as well — no manual start needed after a reboot.
06 — Storage

Choosing Where to Store Photos

💻 Internal Drive (Simplest)

Store photos in a folder like C:\Photos\immich-library. Always available, no risk of a missing drive. Limited by your laptop's built-in storage.

Always available, no plug-in needed
Fastest read/write speed
Limited by laptop SSD size
💾 External Drive (Recommended)

Use an external HDD/SSD like D:\Photos\immich-library. Gives you terabytes of space and is easy to migrate to new hardware later.

Much more storage space
Easy to migrate hardware later
Must be plugged in when Immich runs
🪟
Windows path tip: In the .env file, use backslashes for Windows paths: D:\Photos\immich-library. Docker Desktop on Windows handles the path conversion to Linux format automatically.
07 — Remote Access

Access From Outside Your Home

For the pilot, Tailscale is the easiest option — no domain required, installs like a regular Windows app, and it just works.

🔒 Option A: Tailscale EASIEST

Install Tailscale on your laptop and your phone. They join a private network. Access Immich anywhere via a stable IP — even when your laptop's local IP changes.

Installs as a normal Windows app
No domain, no port forwarding
Free for personal use
Must install on every device you use
🌐 Option B: Cloudflare Tunnel

Get a free domain and use Cloudflare's tunnel client for Windows to expose Immich at https://photos.yourdomain.com. No Tailscale needed on client devices.

Access via any browser, no app needed
HTTPS out of the box
Requires a domain name
Slightly more setup

Option A: Tailscale Setup (Recommended for Pilot)

1

Install Tailscale on your Windows laptop

Download from tailscale.com/download/windows and install. Sign in with a free account (Google/GitHub login works).

2

Get your laptop's Tailscale IP

Click the Tailscale tray icon → it shows your Tailscale IP (starts with 100.x.x.x). Note this down.

3

Install Tailscale on your phone

Install the Tailscale app (iOS/Android), sign in with the same account. Both devices are now on the same private network.

4

Access Immich from anywhere

With Tailscale running on both devices, open Immich at:

http://100.x.x.x:2283
# Replace with your actual Tailscale IP

This works from home, mobile data, abroad — anywhere.

Option B: Cloudflare Tunnel on Windows

1

Download cloudflared for Windows

Download the Windows installer from github.com/cloudflare/cloudflared/releases — get the cloudflared-windows-amd64.msi file and install it.

2

Authenticate and create your tunnel

// PowerShell
cloudflared tunnel login
cloudflared tunnel create immich-tunnel
cloudflared tunnel route dns immich-tunnel photos.yourdomain.com
3

Create config file and run as a Windows service

Create C:\Users\YOU\.cloudflared\config.yml:

tunnel: YOUR-TUNNEL-ID
credentials-file: C:\Users\YOU\.cloudflared\YOUR-TUNNEL-ID.json

ingress:
  - hostname: photos.yourdomain.com
    service: http://localhost:2283
  - service: http_status:404
# Install as a Windows service (auto-starts with Windows)
cloudflared service install
net start cloudflared
08 — Mobile Sync

Auto-Backup From Your Phone

This part is identical to the Linux setup — the Immich mobile app doesn't know or care what OS the server is running.

1

Install the Immich app

Search "Immich" on the App Store (iOS) or Google Play (Android). Install the official free app.

2

Log in with your server URL

If using Tailscale: http://100.x.x.x:2283
If using Cloudflare Tunnel: https://photos.yourdomain.com
If on home WiFi only: http://192.168.1.x:2283 (your laptop's local IP)

3

Enable Background Backup

Profile → Backup → Enable automatic backup. Set WiFi only to avoid mobile data usage. Your full camera roll will sync on the first run — just leave your phone on WiFi overnight.

✓
After this, new photos you take are automatically uploaded to your Immich server in the background — exactly like Google Photos. The app handles it silently.
09 — Availability

Keep the Laptop Running

Your server is only online when the laptop is awake. Configure Windows to stay on as long as it's plugged in:

1

Disable sleep when plugged in

Go to Settings → System → Power & Sleep (Windows 10) or Settings → System → Power (Windows 11).

Under "When plugged in, put my PC to sleep after" — set to Never.

Under "When plugged in, turn off my screen after" — set to Never (or a long time, e.g. 30 min — screen sleep doesn't affect the server).

2

Disable lid-close sleep (if using as a desktop)

Search "Choose what closing the lid does" in the Start menu. Set "When I close the lid → Plugged in" to Do nothing. This lets you close the laptop lid while it keeps running.

3

Set Windows to auto-login after restart (optional)

Run netplwiz in the Start menu → uncheck "Users must enter a username and password". Docker Desktop starts after login, so auto-login ensures Immich comes back up after a power cut without you needing to type a password.

⚠
Only do this if the laptop is in a physically secure location at home. Auto-login means anyone with physical access can use the laptop without a password.
💡 Tip: Check server status remotely

If you're unsure whether your laptop is awake and Immich is running, just try loading http://100.x.x.x:2283 (Tailscale IP) from your phone. If it loads, the server is up. If not, the laptop is likely asleep or off.

10 — Future Migration

Migrating to Proper Hardware Later

When you're ready to move to a dedicated device (Raspberry Pi, mini PC, NAS, etc.), the migration is simple because all your data and config live in two folders.

📦 What to move

Your entire Immich setup lives in these two paths (whatever you set in .env):

C:\immich\                  ← docker-compose.yml + .env config
C:\Photos\immich-library\   ← all your photos and videos
C:\Photos\immich-db\        ← the database

Migration steps (when the time comes)

1

Stop Immich on the Windows laptop

cd C:\immich
docker compose down
2

Copy data to the new device

Copy the photos folder and database folder to the new server. If using an external drive for storage, just plug it into the new device.

# Example using rsync over Tailscale to a new Linux server:
rsync -av "C:/Photos/immich-library/" user@100.x.x.x:/mnt/photos/immich-library/
rsync -av "C:/Photos/immich-db/"      user@100.x.x.x:/mnt/photos/immich-db/
3

Set up Immich on the new device

Follow the main Linux guide on the new hardware, pointing the .env paths to where you copied the data. All photos, albums, users, and face recognition data will be intact.

✓
Your phone's Immich app just needs the server URL updated to the new device's address. Everything else — backup history, albums, all photos — stays exactly as-is.

What the pilot proves

  • Immich works and you like the interface before committing to hardware
  • Remote access via Tailscale or Cloudflare Tunnel is correctly configured
  • Mobile auto-backup is working reliably from your phone(s)
  • You've sized your storage needs (how fast your library grows)
  • You understand the maintenance tasks (updating, monitoring)
