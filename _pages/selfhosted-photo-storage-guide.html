01 — Architecture

How It All Works Together

Your self-hosted photo cloud has three main parts: the server (runs on a home computer/NAS), the tunnel/proxy (provides secure remote access), and the client apps (your phones/tablets that sync photos).

📱 iPhone / Android
💻 Laptop / Browser
↕ HTTPS (encrypted)
🔐 Cloudflare Tunnel
or Tailscale VPN
↕ Secure tunnel
🖥️ Home Server
Immich + Docker
↕ Local disk I/O
💾 HDD / SSD / NAS
Your actual photos
✓
Photos never leave your hardware. The tunnel only carries encrypted traffic — your files stay on your disk at home. Access requires your credentials at all times.

03 — Hardware

What You Need at Home

Server / Computer
Any x86 PC/NAS
Old laptop, mini PC (like Intel NUC), Raspberry Pi 4/5, or a NAS (Synology, QNAP). Needs to stay powered on.
RAM
4 GB minimum
8 GB recommended for face recognition and smooth performance. 2 GB works but slow.
Storage
External HDD/SSD
Any USB or SATA drive works. Size depends on your photo library. 1-4 TB is typical for most families.
OS
Ubuntu / Debian
Ubuntu Server 22.04 LTS is ideal. Windows also works. macOS works but not ideal for 24/7 server use.
Network
Wired Ethernet
Connect your server via Ethernet cable, not WiFi. Much faster uploads/downloads. Home internet upload speed matters for remote use.
Power
UPS (optional)
An uninterruptible power supply protects against power cuts corrupting your database. Strongly recommended.
💡 Budget option

A Raspberry Pi 5 (8GB) + external USB drive is an excellent low-power, always-on server for ~$100–150. It draws only ~5–10W of power, costing about $1–2/month in electricity.

04 — Installation

Setting Up Immich

Immich runs via Docker Compose, which handles all dependencies automatically. Here's the full setup process:

01

Install Docker on your server

Run this on Ubuntu/Debian to install Docker and Docker Compose:

// bash — run on your server
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER
newgrp docker
02

Create the Immich directory and config

mkdir -p ~/immich && cd ~/immich

# Download the official docker-compose file
wget -O docker-compose.yml \
  https://github.com/immich-app/immich/releases/latest/download/docker-compose.yml

# Download the environment template
wget -O .env \
  https://github.com/immich-app/immich/releases/latest/download/example.env
03

Edit the .env file to set your storage path

nano .env

Change these two key lines to point to your hard drive:

# Path where photos will be stored on YOUR disk
UPLOAD_LOCATION=/mnt/photos/immich

# Path for the database
DB_DATA_LOCATION=/mnt/photos/immich-db
⚠
Make sure /mnt/photos points to your external drive. Check with lsblk and mount it first if needed.
04

Start Immich

docker compose up -d

Wait 2–3 minutes for the first launch. Then open a browser on your local network:

http://YOUR-SERVER-IP:2283

Find your server IP with: hostname -I

05

Create your admin account

On first launch, you'll be prompted to create an admin account. Use a strong password — this protects all your photos. Enable 2FA in Settings → Security after setup.

06

Enable auto-start on boot

# Create a systemd service so Immich starts automatically
sudo nano /etc/systemd/system/immich.service
[Unit]
Description=Immich Photo Server
After=docker.service
Requires=docker.service

[Service]
WorkingDirectory=/home/YOUR_USERNAME/immich
ExecStart=/usr/bin/docker compose up
ExecStop=/usr/bin/docker compose down
Restart=always
User=YOUR_USERNAME

[Install]
WantedBy=multi-user.target
sudo systemctl enable immich
sudo systemctl start immich
05 — Remote Access

Access From Anywhere, Securely

You need a way to reach your home server from outside. Two good options:

🌐 Option A: Cloudflare Tunnel

Free, no port forwarding, works with any router. Creates a secure HTTPS URL like photos.yourdomain.com. Best for browser + app access.

✓ Free forever
✓ HTTPS auto-configured
✓ No router config needed
− Requires a free domain
🔒 Option B: Tailscale VPN

Zero-config VPN. Your devices form a private network. Access Immich via its local IP from anywhere. Free for personal use.

✓ Extremely secure
✓ No domain needed
✓ Free for up to 3 users
− Must install on every device

Option A: Cloudflare Tunnel Setup

You need a free domain (get one at Cloudflare, or use an existing one). Then:

1

Install cloudflared on your server

curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb -o cloudflared.deb
sudo dpkg -i cloudflared.deb
2

Authenticate and create tunnel

cloudflared tunnel login
cloudflared tunnel create immich-tunnel
cloudflared tunnel route dns immich-tunnel photos.yourdomain.com
3

Configure the tunnel

nano ~/.cloudflared/config.yml
tunnel: YOUR-TUNNEL-ID
credentials-file: /root/.cloudflared/YOUR-TUNNEL-ID.json

ingress:
  - hostname: photos.yourdomain.com
    service: http://localhost:2283
  - service: http_status:404
cloudflared tunnel run immich-tunnel &

Now https://photos.yourdomain.com securely points to your Immich server at home!

Option B: Tailscale (Easier)

# On your server:
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up

# Then install the Tailscale app on each of your devices
# Access Immich at: http://YOUR-TAILSCALE-IP:2283
06 — Mobile Sync

Auto-Backup From Your Phone

📱 Immich Mobile App

Install the free Immich app from the App Store (iOS) or Google Play (Android). Log in with your server URL and credentials. Enable "Background Backup" — it works exactly like Google Photos auto-backup, uploading new photos automatically over WiFi (or LTE if you choose).

Setup in the app

1

Install and open the Immich app

Search "Immich" on the App Store or Google Play. It's the official app, free.

2

Enter your server URL

Use your Cloudflare URL: https://photos.yourdomain.com
Or Tailscale IP: http://100.x.x.x:2283

3

Enable Background Backup

Go to Profile → Backup → Enable background backup. Set to WiFi-only to avoid using mobile data. Your entire camera roll will upload on first run.

Syncing a computer (optional)

To sync photos from a Windows/Mac/Linux computer, use the Immich CLI:

# Install Node.js first, then:
npm install -g @immich/cli

# Upload a folder of photos:
immich login https://photos.yourdomain.com YOUR_API_KEY
immich upload --recursive /path/to/your/photos
07 — Security

Hardening Your Setup

Security Measure How To Priority
Strong password Use a unique 20+ char password for your Immich account Critical
2FA / TOTP Settings → Security → Enable 2FA in Immich Critical
Keep Immich updated docker compose pull && docker compose up -d Critical
UFW Firewall Block all inbound except SSH. Don't open port 2283 to internet. High
SSH key auth only Disable SSH password login, use key pairs only High
Fail2ban Auto-ban IPs with failed login attempts Medium
Disk encryption Use LUKS to encrypt your photo drive (Linux) Optional
// Quick firewall setup
sudo apt install ufw
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow ssh
# Do NOT open port 2283 — cloudflared handles external access
sudo ufw enable
⚠
Never expose Immich's port 2283 directly to the internet. Always use Cloudflare Tunnel or Tailscale as the secure gateway. Direct exposure without a reverse proxy is a serious security risk.
08 — Backup

The 3-2-1 Backup Strategy

Your home server IS your primary storage — but it can fail. Follow the 3-2-1 rule:

3️⃣ 3 copies

Keep 3 copies of your data total — 1 primary + 2 backups

2️⃣ 2 media

Store on 2 different types of media (e.g., SSD + external HDD)

1️⃣ 1 offsite

Keep 1 copy offsite (another location or cloud) for fire/flood protection

Automated local backup with rsync

# Add to crontab (crontab -e) to backup nightly at 2am:
0 2 * * * rsync -av --delete /mnt/photos/immich/ /mnt/backup-drive/immich-backup/

Free offsite backup options

Backblaze B2
~$6/TB/month
Use rclone to sync your Immich library to B2. Cheapest cloud storage, S3-compatible.
Second Location
Free
A second Raspberry Pi at a family member's house with automated rsync over Tailscale. Truly free offsite backup.
// Backup to Backblaze B2 with rclone
# Install rclone and configure B2:
sudo apt install rclone
rclone config  # follow prompts to add Backblaze B2

# Sync photos to B2 (add to cron for weekly runs):
rclone sync /mnt/photos/immich b2:your-bucket-name/immich --progress
✓
Migration tip: Use Google Takeout to export all your existing Google Photos, then use the Immich CLI or the built-in Google Photos import feature in Settings to bring everything over.
